How it works

Install a lightweight agent, back up to your own S3 bucket, and get restore verification results you can trust.

Step 1

Install the agent

Download a lightweight agent and add a simple config file to your server.

Step 2

Encrypt & upload

The agent creates a database backup, encrypts it locally, and uploads it to your S3 bucket.

Step 3

Verify restores

We periodically download your encrypted backup and run a test restore in a clean environment.

Step 4

See proof anytime

Log into your portal to confirm backups exist and restores have been verified.

Frequently asked questions

Quick answers about backups, encryption, verification, and how your data is handled.

BackupVerified periodically downloads your encrypted backup and performs a test restore in a clean temporary environment. If the restore succeeds, you have proof the backup is usable when you need it.
Backups are encrypted locally first, then uploaded. This reduces risk because raw database data is never sent unencrypted.
You create and control the S3 bucket in your AWS account. BackupVerified uploads into that bucket so you retain full data ownership.
Verification tests can run on a schedule (for example nightly or weekly). The goal is to catch failures early — before an emergency happens.
This site can be deployed by uploading the static HTML/CSS/JS build to Netlify. GitHub is optional.
